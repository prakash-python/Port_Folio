AutoDoc-Validator

AI-Powered Document Intelligence Microservice — A production-grade system designed to extract, validate, and structure financial documents at scale using OCR and Large Language Models.

🚨 Problem Statement

In traditional loan processing systems, document verification is one of the biggest bottlenecks. Loan officers manually verify identity documents, bank statements, and income proofs, which leads to:

  • High processing time (hours to days)
  • Human errors and inconsistencies
  • Difficulty handling multiple document formats
  • High operational cost

Additionally, bank statements from different banks (HDFC, SBI, ICICI, etc.) follow completely different formats, making rule-based parsing extremely unreliable.

💡 Solution Overview

AutoDoc-Validator solves this problem by introducing a fully automated, AI-powered document processing pipeline:

  • Extracts text using OCR
  • Transforms unstructured data into structured JSON using LLM
  • Applies validation rules dynamically
  • Processes documents asynchronously using Celery
  • Returns results via webhook callbacks

The system is built as an independent microservice to handle heavy workloads efficiently.

🏗️ Detailed Architecture Layers

1. Presentation Layer

The React frontend allows users to upload documents and track processing status.

2. API Layer

The Django backend handles:

  • Request validation
  • Job creation
  • Communication with AutoDoc service

3. Queue Layer

Redis acts as a message broker for Celery, ensuring reliable task execution.

4. Processing Layer

Celery workers handle:

  • OCR extraction
  • AI processing
  • Validation logic

5. AI Layer

LLMs are used to convert unstructured OCR text into structured JSON.

6. Integration Layer

Webhook callbacks send results back to the main backend.

⚙️ Key Engineering Decisions

  • Microservice Architecture: Separates heavy workloads from main backend
  • Celery + Redis: Enables asynchronous processing
  • PaddleOCR: High accuracy for Indian documents
  • LLM Integration: Handles multiple formats effectively
  • Webhook Pattern: Eliminates polling
  • Database Locking: Prevents duplicate processing

🚀 Key Features

  • Asynchronous document processing
  • AI-based structured data extraction
  • Supports multiple document types
  • Real-time webhook updates
  • Scalable architecture
  • Error handling and retries

⚡ Challenges & Solutions

Handling Multiple Bank Formats

Solved using LLM instead of rule-based parsing.

Long Processing Time

Handled using Celery async processing.

Concurrency Issues

Solved using database-level locking.

Callback Failures

Implemented retry mechanism.
